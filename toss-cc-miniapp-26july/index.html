01ABOUT

토크크가 무엇인가

토크크는 ‘토스 크리에이터 클럽(Toss Creator Club)’입니다. 매일 유저로 ‘쓰기만’ 하던 토스를, 크리에이터의 눈으로 직접 ‘만드는’ 도구로 바꿔 쓰는 자발적 창작 커뮤니티입니다.

지금은 AI 미니앱으로 시작하지만, 앞으로 토스 안의 다양한 기능을 활용해 창의적인 작업과 이벤트를 함께 열어갑니다. 노션 커뮤니티나 유튜버 커뮤니티처럼 — 같은 걸 만들고 싶은 사람들이 모여, 서로에게 자극이 되고, 피드백을 주고받습니다. 토크크가 가장 중요하게 생각하는 부분입니다.

✦ 왜 혼자가 아니라 같이 하는가

혼자 시작하면, 대부분 흐지부지됩니다.

토크크가 있는 이유는 세 가지입니다. ①같이 하니까 끝까지 만들게 되고, ②막히면 바로 물어볼 동료가 있고, ③결과를 끝까지 함께 지켜보는 팀이 있습니다.

🎯
미니앱 모각코 챌린지란

‘모각코’는 ‘모여서 각자 코딩’입니다. 각자 자기 앱을 만들되, 혼자 끙끙대지 않고 같이 모여 피드백을 주고받으며 한 달 안에 토스에 런칭하는 프로젝트입니다. 결과물은 ‘내 앱’, 과정은 ‘함께’입니다.

클로드 코드 설치

직접 설치하는 것은 클로드 코드 하나입니다. 아래 명령을 한 번만 실행합니다. 나머지 환경은 다음 단계에서 클로드에게 맡깁니다.

코덱스를 쓴다면 npm install -g @openai/codex로 설치합니다. 설치 방법이 막히면 각 도구의 공식 안내를 따릅니다.

4

클로드에게 환경 세팅 시키기 가장 중요

패키지 관리자 설치, 앱인토스 연결 같은 준비는 직접 하지 않습니다. 클로드 코드를 실행한 뒤, 아래처럼 요청하면 클로드가 알아서 처리합니다.

C클로드에게 이렇게 요청합니다
앱인토스 미니앱을 만들 거야. 앱인토스 MCP를 포함해서 필요한 개발 환경을 설치하고 연결해줘.

뒤에서 일어나는 일 — 클로드가 패키지 관리자와 앱인토스 툴킷(ax)을 설치하고, 앱인토스 MCP를 연결합니다.

왜 하는가 — 클로드는 앱인토스만의 규칙(SDK·API)을 모릅니다. 이 연결이 있어야 클로드가 공식 문서와 예제를 참조해 실제로 작동하는 코드를 만듭니다. 없으면 그럴듯하지만 실행되지 않는 코드가 나옵니다.

04WITH CLAUDE

클로드와 잘 만드는 법

클로드가 모든 걸 대신 해주는 것은 아닙니다. 잘 요청할수록 결과가 좋아지고 에러가 줄어듭니다. 실제로 도움되는 것만 모았습니다.

1
앱인토스를 먼저 알려줍니다제일 중요

앞의 4단계에서 MCP를 연결했다면 이 부분은 됐습니다. 여기에 docs-search 스킬을 더하면, 만들려는 기능의 공식 문서를 클로드가 직접 찾아보며 코드를 만듭니다.

docs-search 스킬 설치
/plugin marketplace add toss/apps-in-toss-skills
/plugin install knowledge-skills@apps-in-toss-skills

설치 후엔 이렇게 씁니다 → docs-search로 "인앱 광고 넣는 법" 찾아줘

2
구체적으로 요청합니다

두루뭉술하게 말하면 두루뭉술한 결과가 나옵니다. 화면·동작·조건을 콕 집어 말할수록 정확해집니다.

✕“로그인 만들어줘”
✓“상단에 이메일·비밀번호 입력칸과 로그인 버튼을 넣고, 비어 있으면 ‘입력해 주세요’를 보여줘”
3
화면은 HTML로 먼저 봅니다

글로만 상상하기 어려울 때 “HTML로 먼저 보여줘”라고 하면, 코드에 넣기 전에 화면을 미리 확인하고 고칠 수 있습니다. 완성 후에야 어긋난 걸 발견하는 일을 줄입니다.

4
에러는 그대로 복사해서 붙여넣습니다

직접 해석하려 하지 말고, 에러 메시지 전체를 그대로 클로드에게 보여줍니다. 원인을 찾아 바로 고쳐줍니다.

5
막힌 화면은 캡처해서 보여줍니다

콘솔이나 화면 어딘가에서 막히면, 스크린샷과 함께 “여기서 어떻게 해?”라고 묻습니다. 어디를 누르고 뭘 입력할지 짚어줍니다.

6
모르면 모른다고 말합니다

“무슨 말인지 모르겠어. 뭘 하면 돼?”라고 하면, 다음에 할 일을 더 쉬운 말로 다시 안내받습니다.

기억할 것

맥락 주고 → 결과 보고 → 다시 요청하면서 같이 만드는 과정입니다. 핵심은 클로드가 앱인토스를 이해한 상태로 짜게 만드는 것입니다.

06FAQ

자주 묻는 질문

시작 전에 많이 나오는 질문만 모았습니다. 질문을 누르면 답이 펼쳐집니다.

Q꼭 여름 주제여야 하나요?⌄
아닙니다. 토크크 미션 자체는 자유주제라 꼭 여름이 아니어도 됩니다. 다만 토스 공식 7월 챌린지(‘미니앱과 함께하는 여름’, ~7/29)에 출품한다면, 그때는 여름 주제에 맞춰야 심사 대상이 됩니다.
Q클로드 코드 말고 챗지피티(Codex)를 써도 되나요?⌄
됩니다. 앱인토스 개발자센터가 클로드 코드와 코덱스를 모두 안내합니다. 명령어만 각 도구에 맞게 쓰면 되고, 윈도우에서도 가능합니다. 이 가이드는 클로드 코드 기준으로 정리했습니다.
Q코드를 짰는데 실행하면 자꾸 에러가 납니다.⌄
십중팔구, 클로드가 앱인토스 규칙을 모른 채 코드를 짜서 그렇습니다. 실력이 없는 게 아니라 맥락이 없는 것입니다. 4단계의 MCP 연결이나 docs-search 스킬을 설치하면 크게 줄어듭니다.
Q유료 결제를 꼭 해야 하나요? 몇 개 만들어야 하나요?⌄
무료 버전으로 먼저 감을 잡아보고, 필요하면 유료로 넘어가는 것을 권합니다. 개수는 기본 1인 1앱, 더 만들 수 있으면 얼마든지 환영합니다.
Q답을 아는 사람이 없는 질문이면 어떻게 하나요?⌄
그럴 때는 채널에 올려 함께 찾아봅니다. 일방적으로 묻기보다 서로 답을 찾아나간다는 마음으로 질문하고 답하는 것이, 가장 토크크다운 방식입니다.
